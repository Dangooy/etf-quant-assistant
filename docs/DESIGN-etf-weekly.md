# ETF 周度配置系统设计方案

> 2026-07-06 与用户确认的需求基线，实施前的唯一依据。改本文件 = 改需求，需人工确认。

## 一、需求基线（已确认）

| 维度 | 决定 |
|---|---|
| 目标 | **年化 5-6%**（2026-07-07 经 Phase 3 回测验证后用户确认写实：完整规则 2016-2026 年化 5.11%、最大回撤 6.21%、卡玛 0.82；原 6-8% 目标在 ≤10% 回撤约束下不成立。保留趋势过滤作为极端熊市保险，拒绝了关闭趋势过滤换 6.13% 的方案） |
| 回撤底线 | 组合自高点回撤 ≤10%，这是**硬约束**，优先级高于收益 |
| 标的 | 场内 ETF 全广谱：宽基 + 红利低波 + 债券 + 黄金 + QDII |
| 节奏 | 周度：每周五收盘后系统自动分析，周末推送，下周一执行 |
| 输出 | 可直接执行的交易清单（代码/名称/买卖/份额/参考价/理由） |
| 存量 | 现有存量持仓纳入管理，按规则分批迁移（8-12 周过渡） |
| 边界 | 无实盘接口，下单永远人工；agent 不得基于新闻情绪偏离规则 |

## 二、策略设计

### 2.1 资产池（全部场内，akshare 免费数据覆盖）

### 2.2 目标权重（战略中枢）

- 防守层 ≥55%：国债 35% + 短融 20%
- 权益层 ≤35%：A股腿（300/红利/500 中按动量取 2 只，各 ~12.5%）+ 海外腿（纳指/标普取 1 只，~10%）
- 黄金 10%

中枢对应的静态预期约 5-6%，6-8% 的差额来自趋势过滤避开熊市 + 动量轮动的超额。**具体参数以 Phase 3 回测结果为准，回测不达标就下调收益预期，不放大风险。**

### 2.3 周度规则（全部机械可执行）

1. **趋势过滤**：任一风险腿收盘 < 200 日均线 → 该腿权重归零，资金回短融ETF；重新站上 → 恢复。
2. **动量轮动**：A股三只按 26 周收益率排序持前 2；海外两只持动量高者。
3. **再平衡带**：实际权重偏离目标 ±20%（相对值）以上才生成交易，否则不动（控制交易频率与成本）。
4. **回撤断路器**（组合级，日频监控、周频执行，极端情况当日告警）：
   - 自高点回撤 ≥6%：全部风险腿权重减半；
   - ≥8%：风险腿清零，只留债+短融+黄金；
   - 恢复条件：对应标的重回 200 日线上方，按规则逐步重建。
   - risk_zero 触发即重置高水位，恢复由趋势规则决定；极端连续触发情形下累计回撤可能超过 10%（单轮上限 8%，趋势过滤使连续触发罕见）。
5. **QDII 溢价保护**：纳指/标普 ETF 溢价 >3% 时暂停买入该标的（IOPV 数据 akshare 可取，取不到时人工提示核对）。

### 2.4 存量迁移规则

- 目标池外标的全部逐步清出，**卖出顺序由规则定，不由盈亏情绪定**：先减超过单只上限的仓位，再减其余，每周卖出总额 ≤ 总资产 15%。
- 卖出资金先进短融ETF，权益腿按 2.3 的信号逐步建仓，不许"卖完当周满仓买"。
- 预计 8-12 周完成过渡；期间现有风控规则继续对存量生效。

## 三、工程实施（分五期）

### Phase 1：数据与账户模型
- `config.py`：加 `ETF_POOL`、`TARGET_WEIGHTS`、策略参数区（均线周期/动量窗口/再平衡带/断路器阈值）。
- `portfolio.json` 加 `cash` 字段；权重全部改为占总资产（含现金）。
- fetcher：日线聚合周线（resample，不新增数据源）；ETF 池标的的历史数据首次全量拉取入缓存（回测需要 2015 至今）。

### Phase 2：配置引擎与交易清单
- 新模块 `allocation/`：输入行情+当前持仓 → 输出目标权重（趋势过滤→动量→断路器）。
- 新模块 `rebalance/`：目标权重 vs 当前持仓 → 交易清单（100 份取整、单笔 <2000 元忽略、费用估算、每条附规则理由）。
- 迁移模式：目标池外持仓按 2.4 生成逐周卖出清单。

### Phase 3：组合级回测（验收关卡）
- 扩展回测引擎支持多标的+权重调仓（现引擎只支持单标的）。
- **数据口径（2026-07-07 经用户确认变更）**：回测使用天天基金**累计净值**序列（akshare `fund_etf_fund_info_em`，域名未被封禁）——含分红、历史不随除权漂移、可复现，优于 qfq 价格；日常周度信号仍用场内价格。两者差异为 ETF 折溢价噪声（±0.5% 以内），回测报告中必须标注此口径。
- 用 2015-至今数据回测完整规则，**验收标准：年化 ≥6%，最大回撤 ≤12%（回测口径），2018/2022 年回撤显著小于沪深300**。
- 不达标 → 调参数或下调预期，回测报告留档 `data/reports/`。

### Phase 4：周报 CLI + 定时值守
- `python -m quant_assistant weekly`：组合状态、回撤水位、各腿信号、交易清单、迁移进度。
- launchd 每周五 16:00 自动跑，结果写 `data/reports/weekly-<date>.md`；断路器触发时当日推送告警。

### Phase 5：测试固化
- 本轮及上轮的验证脚本转为 pytest（撮合、费用、断路器、清单取整、迁移限速）。
- CLAUDE.md 增补 weekly 命令说明与纪律条款。

## 四、纪律条款

1. 交易清单只能由规则生成；agent 的定性分析（新闻/财报）只能作为附注，不得改动清单。
2. 改参数 = 改 `config.py` 并在 git commit 中写理由，一周最多改一次。
3. 断路器触发时，唯一正确操作是执行清单，不许"再等等看"。
4. 连续 4 周未执行清单，系统在周报中黄字提醒偏离度。
